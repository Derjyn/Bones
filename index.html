Bones Grid Usage and Demo

Explanation

Bones is a minimal-frills grid system utilizing flexbox. There are a few extras included, such as a basic reset, some utility classes, a clean print style, and a few media queries for common device screen sizes. The goal is to keep Bones as bloat-free and customizable as possible, while still offering a powerful foundation for layout development.

Spine

A spine is the parent flexbox container which may contain many or a single rib. They are laid out in rows (not very similar to a spine, I know) and can fill their parent containers in width. There are a couple of utility classes that can be slapped on the body to change the width and center everything, using a couple of common widths: nine-sixty and twelve-eighty. In it's current state, Bones doesn't support nested spines. It's on the roadmap though!

Ribs

Ribs make up the items within a flexbox container. In the Bones system, each rib is sized in eigths. So a 1/8 sized rib would use the class rib one, a 2/8th sized rib rib two, and so on up to a 8/8 size rib, which is rib full. There are also useful utility classes that can help tweak layouts, outlined below.

Utilities

There are several utility classes included with Bones that allows for further adjustments to layouts:

  • l-margin, r-margin, t-margin and b-margin
  • breaker
  • filler
  • grower

l-margin, r-margin, t-margin and b-margin will add a 16 pixel margin to the left, right, top or bottom of a rib. Useful for adding in gutter-like functionality manually. breaker slams in a sort of seperator that is full width, and has varying heights (breaker sixteen for a 16 pixel height for example). For the purposes of this demo, a breaker follows the rib sizes example below, and is styled (normally it would be invisible, if one so desired). filler helps to fill up the space left over in a row with ribs, so a new row of ribs doesn't smash up against the others. grower will force a rib to grow in width, to the remaining space within a spine. This helps when you don't specify the size of a rib, but want it to stretch out. On smaller screens (such as that of an iPhone 4), sizes of margins and other spacing drops from 16 pixel increments, to 8 pixel increments.
