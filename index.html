Bones Grid Demo and Usage

Explanation

Bones has a minimal-frills grid system utilizing flexbox. There are a few extras included, such as a basic reset, some utility classes, a clean print style, and a few media queries for common device screen sizes. The goal is to keep Bones as bloat-free and customizable as possible, while still offering a powerful foundation for layout development. Over time, various other goodies may find their way into the Bones framework

Spine

A "spine" is the parent flexbox container, which can contain "ribs". They are laid out in rows (not very similar to a spine, I know), and are fill their parent containers. In this demo, the body is the parent container is full width. There are a couple of utility classes that can be slapped on the body to change the width and center everything, using a couple of common widths: "nine-sixty" and "twelve-eighty". In it's current state, Bones doesn't support nested spines. It's on the roadmap though!

Ribs

"Ribs" make up the items within a flexbox container. In the Bones system, each rib is sized in eigths. So a 1/8 sized rib would us the class "rib one-eigth", a 2/8th sized rib "rib two-eigths", and so on up to a 8/8 size rib, which is "rib full". There are useful utility classes that can help tweak layouts outlined below.

Utilities

There are several utility classes included with Bones that allows for further adjustments to layouts:

  • l-margin and r-margin
  • spacer
  • grower
  • breaker

l-margin and r-margin will add a 16 pixel margin to the left or right of a rib. Useful for adding in a gutter manually. Spacer, when added to a rib will add a 16 pixel margin below the rib, handy when you want to add some space between a row of ribs. Grower will force a rib to grow in width, to the remaining space within a spine. This helps when you don't specify the size of a rib, but want it to stretch out. Breaker slams in a sort of seperator that is full width, and is 64 pixels in height. For the purposes of this demo, a breaker follows the rib sizes example below, and is styled (normally it would be invisible, if one so desired).
